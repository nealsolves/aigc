03 Key Boundaries
Beta Scope

These diagrams describe the source-only v0.9.0 beta surface on local develop. The installable package remains v0.3.3.

Workflow Layer

AIGC.open_session(), GovernanceSession, and SessionPreCallResult are the real beta additions shown here. They wrap the invocation kernel; they do not replace it.

Kernel Invariant

The ordered invocation gates stay intact in the beta line. Workflow governance constrains when a step may run, then the same authorization and output-side checks still execute.

Separate Evidence

Invocation artifacts remain one per invocation attempt. Workflow artifacts are separate session outputs, with invocation-correlation metadata added rather than replacing the invocation contract.

Beta CLI

The diagrams include the shipped beta adoption surface: aigc workflow init, aigc workflow lint, and aigc workflow doctor.

Planned-Only Exclusions

AgentIdentity, AgentCapabilityManifest, public ValidatorHook, BedrockTraceAdapter, A2AAdapter, and workflow trace/export stay out of these beta diagrams.

Session Token Boundary

SessionPreCallResult is workflow-bound. It must be completed by the owning GovernanceSession, not by the module-level enforce_post_call() path.

Ownership Boundary

The host still owns orchestration, provider SDKs, transport, retries, credentials, and tool execution. AIGC governs, blocks, and emits evidence.
