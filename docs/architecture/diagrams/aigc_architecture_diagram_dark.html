03 Key Boundaries
Decorator Modes

@governed defaults to unified post-call enforcement for backward compatibility. Set pre_call_enforcement=True to run Phase A before the wrapped function and Phase B after it returns.

Phase A / Phase B

Split mode moves the model call boundary between post_authorization and pre_output. Unified mode still evaluates the same ordered gates in one enforcement call.

Audit Chain

AuditChain is not part of the automatic enforcement pipeline. It is an opt-in utility the host applies to artifacts after enforcement.

Compliance Export

aigc compliance export is an offline analysis step over stored audit artifacts, not a live runtime gate.

Pre-Pipeline Failures

Pre-pipeline failures produce schema-valid FAIL artifacts with policy_version: "unknown", but bypass the core gate sequence.

Signing Semantics

If signing is configured, the same audit artifact is signed before sink emission and before it is returned to the caller. PASS and FAIL artifacts can both carry signatures.

Async + Instance APIs

enforce_invocation_async(), enforce_pre_call_async(), enforce_post_call_async(), and the matching AIGC instance methods ship in the v0.3.2 runtime.
