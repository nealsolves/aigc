03 Key Boundaries
Decorator Modes

@governed defaults to split enforcement since v0.3.3. Phase A runs before the wrapped function executes and Phase B runs after it returns. Pass pre_call_enforcement=False for legacy unified mode (deprecated).

Phase A / Phase B

Split mode moves the model call boundary between post_authorization and pre_output. Unified mode still evaluates the same ordered gates in one enforcement call.

ProvenanceGate

ProvenanceGate is a built-in gate registered at pre_output. It blocks output when context.provenance.source_ids is missing or empty.

Artifact Provenance

Every PASS or FAIL artifact can carry v1.4 provenance fields: source_ids, derived_from_audit_checksums, and compilation_source_hash.

Audit Chain

AuditChain is not part of the automatic enforcement pipeline. It is an opt-in utility the host applies to artifacts after enforcement.

Lineage Analysis

AuditLineage reconstructs parent and child relationships from stored audit trails, and aigc compliance export --lineage exposes that analysis offline.

Pre-Pipeline Failures

Pre-pipeline failures produce schema-valid FAIL artifacts with policy_version: "unknown", but bypass the core gate sequence.

RiskHistory

RiskHistory is an advisory utility over emitted risk scores. It classifies trend direction over time and does not alter gate order or blocking semantics.

Signing Semantics

If signing is configured, the same audit artifact is signed before sink emission and before it is returned to the caller. PASS and FAIL artifacts can both carry signatures.

Async + Instance APIs

enforce_invocation_async(), enforce_pre_call_async(), enforce_post_call_async(), and the matching AIGC instance methods ship in the v0.3.3 runtime.
