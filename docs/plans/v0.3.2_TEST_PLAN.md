# v0.3.2 Test Plan — Split Enforcement

Date: 2026-04-04
Status: Proposed
Baseline suite: 597 tests (`v0.3.1`)

---

## 1. Test Strategy

Goals:

- validate split phase behavior end-to-end,
- preserve unified backward compatibility,
- preserve artifact schema and replay guarantees,
- prove no gate-order regressions,
- prove safe `PreCallResult` lifecycle.

Execution gates:

1. New split tests must pass.
2. Existing test suite must remain green.
3. Golden replay snapshots must be stable on non-volatile fields.

## 3. PreCallResult Test Plan

Coverage targets:

- immutability of top-level object and nested structures,
- serialization and pickling safety,
- one-time consumption enforcement,
- missing/reused object failures.

Test cases:

- PCR-01 frozen dataclass rejects attribute assignment.
- PCR-02 nested mapping/list mutation attempt fails.
- PCR-03 `pickle.dumps/loads` roundtrip retains equality.
- PCR-04 deterministic `to_dict` (if provided).
- PCR-05 single-threaded first consumption succeeds and flips internal state.
- PCR-06 missing pre-result rejected by `enforce_post_call`.
- PCR-07 same pre-result reused twice fails closed on second use.
- PCR-08 pickle roundtrip preserves consumed vs unconsumed lifecycle state.

---

## 4. @governed Test Plan

Modes:

- `pre_call_enforcement=False` (default unified behavior)
- `pre_call_enforcement=True` (split)

Sync and async coverage required for each mode.

Core scenarios:

- GOV-01 default mode calls function first, then unified enforcement.
- GOV-02 split mode blocks function when Phase A fails.
- GOV-03 split mode calls function and enforces post-call on pass.
- GOV-04 schema fail in split mode raises after function return.
- GOV-05 argument extraction compatibility unchanged (`input_data`, `input`, reordered params, kwargs).
- GOV-06 non-dict coercion behavior remains compatible.

1. Phase A PASS -> Phase B PASS.
2. Phase A FAIL role unauthorized.
3. Phase A FAIL precondition missing.

6. Phase A PASS -> Phase B FAIL strict risk threshold.
