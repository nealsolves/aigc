# AIGC Workflow Starter Index

Starters are generated by `aigc workflow init --profile <profile>` and provide
a working governed workflow for each use case. All starters run without external
API keys, Bedrock credentials, or A2A setup.

## Available profiles

| Profile | Generated by | Key features | Generated files |
|---------|-------------|--------------|-----------------|
| `minimal` | `aigc workflow init --profile minimal` | 2-step governed workflow, no external API needed | `policy.yaml`, `workflow_example.py`, `README.md` |
| `standard` | `aigc workflow init --profile standard` | 3-step workflow with approval checkpoint (pause/resume) | `policy.yaml`, `workflow_example.py`, `README.md` |
| `regulated-high-assurance` | `aigc workflow init --profile regulated-high-assurance` | `ProvenanceGate` (source_ids required), tool budget enforcement | `policy.yaml`, `workflow_example.py`, `README.md` |

## Notes

- All starters generate into a subdirectory named after the profile (e.g.,
  `governance/`, `standard-starter/`, `regulated-starter/`).
- Run `python workflow_example.py` inside the generated directory to exercise
  the full governance lifecycle immediately after generation.
- Use `aigc workflow doctor <generated-dir>/` if you modify a starter and want
  structured diagnostics.
- See [STARTER_RECIPES.md](STARTER_RECIPES.md) for per-profile instructions,
  expected output, and customization notes.
